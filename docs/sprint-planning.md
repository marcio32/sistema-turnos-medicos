## Definition of Done (DoD)

Una historia se considera **DONE** cuando cumple TODOS los siguientes criterios:

- [ ] El código está escrito y cumple los criterios de aceptación de la historia
- [ ] El código pasa el linting (ESLint + Prettier) sin errores ni warnings
- [ ] Los tests unitarios están escritos y pasan (cobertura mínima del componente/servicio)
- [ ] El código fue revisado por al menos un compañero (Code Review aprobado)
- [ ] El PR pasa el pipeline de CI (lint + build + test)
- [ ] La funcionalidad fue probada manualmente en el entorno local
- [ ] La documentación fue actualizada si corresponde (endpoints, componentes nuevos)
- [ ] No hay deuda técnica intencionalmente introducida sin ticket asociado
